# 市场异动：自选操作与详细说明设计

## 目标

在市场异动页面的每个股票事件上提供就近的“加入自选”操作，并让详情面板同时展示可读的结构化解读和数据源提供的原始异动说明。

## 范围

- 市场异动列表每条事件提供图标按钮；详情头部提供带文字的同等操作。
- 复用 `useWatchlistStore` 的 `stocks`、`refresh` 和 `add`，以及项目既有 toast 反馈。
- 针对涨停、上涨、下跌、跌停/炸板、成交/资金和其他异动，生成“信号含义”“盘面解读”“关注风险”三部分说明。
- 保留 `MarketPulseItem.info`，标为“原始异动说明”。

不改变后端异动接口、市场异动分类逻辑或自选股接口。

## 交互与状态

1. 组件挂载时刷新自选股列表，按股票代码判断当前事件是否已加入自选。
2. 用户可在列表项或详情头部点击加入自选；两个入口共享同一股票代码的提交状态，提交期间均不可重复触发。
3. 加入成功后，显示成功提示并刷新后的自选状态；接口返回已存在时，显示“已自选”状态，不重复提示成功。
4. 加入失败时，显示错误提示，按钮恢复可用；不影响事件选择、筛选或自动刷新。
5. 自选状态未知或列表加载中时，操作入口保持可用，API 的幂等返回仍作为最终结果。

## 说明生成规则

结构化说明只基于已有 `changeType` 和 `info`，不推断价格、成交量、消息面或因果关系：

- 涨停异动：强调封板或冲板强度，提示封单变化、炸板与次日溢价的不确定性。
- 上涨异动：说明短时上行或买盘增强，提示追高风险与量价持续性。
- 下跌异动：说明短时走弱或卖压增加，提示支撑位、流动性和情绪扩散风险。
- 跌停/炸板：说明强势价格状态被打破或下行压力，提示封单、承接与波动扩大风险。
- 成交/资金：说明成交或资金活动出现异常，提示单一时点信号不足以确认趋势。
- 其他：说明检测到未归类的盘中变化，提示结合行情和公告进一步验证。

每部分使用条件性、非建议性措辞；页面不构成投资建议。

## 实现边界

- 修改 `frontend/src/components/market-pulse/MarketPulsePanel.tsx`，并在同目录新增纯函数模块（若需要）以集中管理说明内容，避免把规则散落在 JSX 内。
- 修改或新增同目录 Vitest 测试，mock 自选股 store 和 toast，与现有测试隔离。
- 新增的文案通过现有 `t(key, fallback)` 模式提供中文 fallback，不要求本次同步扩展所有语言包。

## 验收与测试

- 未自选股票在列表与详情均能加入，成功后两处均显示已自选。
- 已自选股票不能重复提交。
- 接口异常会显示失败反馈且恢复操作能力。
- 每个主要异动类别渲染对应的信号含义、盘面解读和关注风险，并保留原始说明。
- 运行市场异动组件单测和 `npm run build`。
